Deployment & Integration Guide

1. Automated Bootstrap & Installation

Run the quick onboarding script from your local terminal to instantly pull down the core logic, templates, and container configurations from the repository.

Option A: Windows (PowerShell) 
# Download and execute the interactive bootstrapper
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/rjmad1/RajaJeevanLoopEngineering/main/interactive-bootstrap.ps1" -OutFile "interactive-bootstrap.ps1"
.\interactive-bootstrap.ps1
Option B: Linux / macOS (Bash) 
# Download and execute the cross-platform setup script
curl -O https://raw.githubusercontent.com/rjmad1/RajaJeevanLoopEngineering/main/bootstrap.py
python3 bootstrap.py

Note: The script will prompt you for your project path and target loop track (Greenfield, Brownfield, Modernization, or Incident Response) to automatically scaffold the workspace.

2. Activate the State Engine

Run the containerized Java state engine to manage circuit breaking, phase tracking, and state validation transitions:

# Build the target engine image
docker build -t loop-engine https://github.com/rjmad1/RajaJeevanLoopEngineering.git#main

# Start the engine (exposes the state management API on port 8080)
docker run -p 8080:8080 loop-engine

3. IDE Prompt Insertion

Save the following minimalist configuration block as a .cursorrules or .github/copilot-instructions.md file at the root of your project to control agent behaviors:

# SYSTEM INSTRUCTION: ENFORCE BOUNDED LOOP ENGINEERING
You must execute all workspace updates inside the formal boundaries of the RajaJeevanLoopEngineering framework.

## 1. State-Bound Lifecycle
You cannot generate arbitrary code without stepping sequentially through these explicit phases:
Discovery ➔ Planning ➔ Implementation ➔ Verification ➔ Reflection.

## 2. Mandatory Local Execution Commands
Before moving between phases, you must validate transitions using the agnostic state utilities:
* Register Phase Transition: ./loop-control.sh transit <loop-id> <CURRENT_PHASE> <NEXT_PHASE>
* Check Circuit Breaker Status: ./loop-control.sh status <loop-id>

## 3. Core Constraints
* Never execute continuous code changes if verification tests fail iteratively.
* Stop immediately if the loop-control circuit breaker trips.
